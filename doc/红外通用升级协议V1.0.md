## 协议格式定义

|                     | **包头** |          |         |          |          |          | **数据** |          |
| ------------------- | -------- | -------- | ------- | -------- | -------- | -------- | -------- | -------- |
|                     | 起始符   | 设备地址 | 命令号  | 操作类型 | 数据长度 | CRC16    | 数据     | CRC16    |
| **数据 <br />类型** | uint8_t  | uint16_t | uint8_t | uint8_t  | uint16_t | uint16_t |          | uint16_t |
| **字节<br />长度**  | 1        | 2        | 1       | 1        | 2        | 2        | 不定长   | 2        |
| **范围**            | #        | 0-65535  | 0-255   | 0-10     | 0-65535  | 0-65535  |          | 0-65535  |

注：

- 此包头并非TCP包头，是传输数据中自定的识别设备地址及命令的数据段。

- 本协议起始符固定为字符 ’#’，十六进制为0x23。

- 操作类型定义，0为基础读，1为基础写，10为消息确认，2-5为自定义读，6-9为自定义写。

- 数据长度仅为数据部分长度，不包含数据后的CRC。

- 数据区中的所有uint16_t或uint32_t数据，先发送低字节。

- 协议中第一个CRC16是对包头的校验，第二个CRC16是对数据的校验，且CRC16采用业界通用的ModBus RTU CRC算法。

- 协议包头数据结构和协议数据部分1字节对齐。如下示例：

  ```c
  /* 包头结构体 */
  typedef __packed struct {
      uint8_t  sop;      /* '#' */
      uint16_t dev_id;   /* 设备地址 */
      uint8_t  cmd;      /* 命令 */
      uint8_t  op_type;  /* 操作类型 */
      uint16_t data_len; /* 载荷数据长度（字节数，不包括CRC16） */
      uint16_t crc;      /* 包头CRC16 */
  } prot_tp1_head_t;
  ```

  ## 设备地址定义

  **0**：代表广播地址（注意广播时不应答）

  **1**：代表主地址

  **2、3、4、5**：分别代表4块从地址

  

  **注意：**示例表中黑体的数据为**变化项**



## 固件升级协议

### 50号命令 ： **升级通知（上位机3次重试）**

（1）若版本号符合升级要求，设备将升级确认

| 时序 | 地址     | 命令号 | 操作类型 | 数据长度 | 数据                                                    |
| ---- | -------- | ------ | -------- | -------- | ------------------------------------------------------- |
| 平台 | 设备地址 | 50     | 1        | 15       | 软件版本号+数据包总大小+单包大小+数据包总数+数据包CRC32 |
| 设备 | 设备地址 | 50     | 1        | 15       | 软件版本号+数据包总大小+单包大小+数据包总数+数据包CRC32 |

示例：

| 源头   | #    | 地址 | 命令号 | 操作类型 | 数据长度 | 包头CRC16 | 数据(版本号 总大小82436 单包大小1024 包总数80 CRC32) | 数据CRC16 |
| ------ | ---- | ---- | ------ | -------- | -------- | --------- | ---------------------------------------------------- | --------- |
| 平台   | 23   | 0100 | 32     | 01       | 0F00     | 529B      | 010102 04420100 0400 5000 E17E8252                   | C2 95     |
| 主控板 | 23   | 0100 | 32     | 01       | 0F00     | 529B      | 010102 04420100 0400 5000 E17E8252                   | C2 95     |

（2）若版本号不符合升级要求，设备将拒绝升级

| 时序 | 地址     | 命令号 | 操作类型 | 数据长度 | 数据                                                    |
| ---- | -------- | ------ | -------- | -------- | ------------------------------------------------------- |
| 平台 | 设备地址 | 50     | 1        | 15       | 软件版本号+数据包总大小+单包大小+数据包总数+数据包CRC32 |
| 设备 | 设备地址 | 50     | 1        | 0        | 无（给文本提示）                                        |

示例：

| 源头   | #    | 地址  | 命令号 | 操作类型 | 数据长度 | 包头CRC16 | 数据(版本号 总大小82436 单包大小1024 包总数80 CRC32) | 数据CRC16 |
| ------ | ---- | ----- | ------ | -------- | -------- | --------- | ---------------------------------------------------- | --------- |
| 平台   | 23   | 01 00 | 32     | 01       | 0F 00    | 52 9B     | 010102 04420100 0400 5000 15AB E17E8252              | B7 79     |
| 主控板 | 23   | 01 00 | 32     | 01       | 00 00    | 52 9B     |                                                      |           |

在正式开始升级数据传输前，平台与设备进行升级信息的确认，包含：

* 升级软件版本号(版本号由主要版本号、次要版本号、修正版本号构成)
* 数据包总大小
* 单包数据大小
* 数据包总数
* 数据包 CRC32
* 升级确认

  * 升级软件版本号大于设备软件版本号（断包续传注意必须是同一bin文件）
  * 主控芯片一致（硬件版本号一致）
* 拒绝升级

  * 升级软件版本号小于等于设备软件版本号
  * 主控芯片或硬件不一致

数据结构解析：

|              | 数据类型  | 长度 | 范围     |
| ------------ | --------- | ---- | -------- |
| 软件版本号   | uint8_t*3 | 3    |          |
| 数据包总大小 | uint32_t  | 4    | 1~262144 |
| 单包数据大小 | uint16_t  | 2    | 1-65535  |
| 数据包总数   | uint16_t  | 2    | 1-65535  |
| 数据包 CRC32 | uint32_t  | 4    | 1~262144 |
